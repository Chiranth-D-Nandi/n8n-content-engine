Connection Debugging
What gets tested:
  • ✓ Flask API health check (port 5000) — keeps the job queue running
  • ✓ n8n webhook URL — where pipeline requests are sent

If Flask is offline:
  • Run: python api.py in your matiks folder
  • Check: curl http://localhost:5000/health

If n8n webhook fails:
  • Is n8n running? Run: n8n start
  • Does your webhook exist? Check n8n → Workflows → [Your workflow]
  • Did you set method=POST on your Webhook node?
  • CORS issue? Add to Webhook response headers: Access-Control-Allow-Origin: *
  • Test directly: curl -X POST http://localhost:5678/webhook/matiks-reel -H "Content-Type: application/json" -d "{\"test\":true}"
